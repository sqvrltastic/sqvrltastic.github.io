A guide to self-hosting and optimizing your website's fonts (+ some typography tips)

Published: 04/22/2026

Table of contents:

  1. Why should you self-host fonts in the first place?
  2. Where to get web fonts for free?
  3. Introduction to subsetting fonts
  4. Subsetting fonts using Google Font Helper
  5. Subsetting fonts using FontTools and Brotli
  6. Adding custom fonts to CSS stylesheet
  7. Bonus typography tips

Why should you self-host fonts in the first place?

Here are some of the reasons you should self-host your fonts, instead of relying on 3rd party CDNs:

  • Improved user privacy; If you are importing fonts from a 3rd party service such as Google for example, your users' data is beholden to their own privacy policy and you might be breaking GDPR.
  • If the 3rd party service for whatever reason becomes unavailable, your custom fonts will be unavailable as well.
  • Self-hosting method is marginally faster and more performant. Now, if you are like me and have been doing web dev for a while, you might have heard that linking fonts from Google is faster because of a thing called "caching", but it is not the case anymore. Most browsers now have a separate cache for every website, this practice is called "cache partitioning".
  • Self-hosting fonts is the easiest way to ensure that your website's text looks consistent across different browsers and operating systems. This is not the case if you're relying on system fonts, which do, however, have the advantage in the performance and simplicity department.

Where to get web fonts for free?

Important: Make sure you comply with the font's license for your intended use!!! Please don't just grab random fonts you downloaded online and upload them to your website. If you don't have a commercial license for your font and you get caught red-handed it will tarnish your reputation and might even get you into legal trouble. Either buy a license or only use fonts that are free for commercial use (all Google fonts are, linked below).

Probably the most well-known option, used by both hobbyists and professionals, is Google Fonts. Other popular options are 1001fonts and Dafont, but please check the author's README file before using them in your own projects!

Note on web font formats

When you download a font to your PC, it will probably be in .ttf (TrueType) format. TTF fonts are NOT optimized for web use! While they will work if you upload and link them in your CSS sheet, they are usually too large and will negatively affect your website loading times, sometimes significantly! The current file standard for web fonts is WOFF2 and it's widely accepted. There is also basic WOFF if you need to support some very old browsers, and I will actually show you later how to add it as a fallback in case you want to! I will show you how to convert .ttf to .woff2 font files below...

Introduction to subsetting fonts

Say you have a font file that supports 5 different languages, extended Latin, Cyrillic and Greek alphabet, but you only need the letters used in the English language? Or maybe, your font comes with 10 different weights and italics variants, but you only need the regular weight, bold, italic and bold-italic variants? You cut everything else out. That's called "subsetting". Subsetting creates a smaller font file that can drastically improve loading times, which is super important for websites and web applications. I will show you two different methods I use below. Also, I will mention FontSquirell here because I know some people like it, but personally, I don't use it, so I can't vouch for it. Use it at your discretion.

Subsetting using Google Fonts Helper

The first method is by far the easiest, but it only works with Google Fonts. Go to Google Fonts and look for a font that you like. There are some filters and tags that you can use to refine your search. Make sure your chosen font supports languages and symbols you intend to use on your website. You can download and install the font on your PC to play around with it, but it's not necessary.

Next, go to the website Google webfonts helper. Search for a font that you just downloaded

In the "3. Copy CSS" section just leave everything as default ("Modern Browsers"). Scroll a bit more down and you will have your newly optimized font files waiting for you. Just download the .zip archive and extract your new .woff2 files somewhere. Skip to >>Adding custom-fonts to CSS stylesheet if you just want to learn how to self-host it ASAP.

Subsetting using FontTools and Brotli

The second method is a bit more complicated, but also more versatile and can be used with absolutely any font. Now, since I'm on Windows 11 (the price I pay for being a gaymer), I wrote this part primarily with Windows users in mind, but the process shouldn't be too different on other operating systems.

You will need to have preinstalled:

  • Git Bash
  • Python 🐍

If you don't know how to install either, just look up the most up-to-date tutorial online, depending on your OS. During the Python installation, make sure to check ✅ the "Add python.exe to PATH" option during the installation (on Windows). To check if Python is installed correctly in Git Bash, run python --version, it should return a version number (e.g. "Python 3.12.1"). After you're done with that, run python -m ensurepip to check if you have pip (we will need it for the next step).

Next, you will need to install FontTools and Brotli. To do so, open Git Bash and install it with pip. Run: python -m pip install fonttools brotli (this should work, regardless of your shell configuration). Brotli is required when processing woff2 files, or else FontTools will throw an error.

For demonstration purposes, I will be using the same font that I'm using on this website — Inter; you can download it from the aforementioned Google Fonts if you want to follow along. It supports a lot of languages, weights (thin vs bold), has true italics (some fonts come without a separate italics font, so browsers just simulate it by skewing the letters) and a few different optical size presets (optical sizing in fonts adjusts the shape of letters and spacing for better look and legibility at different font sizes, like small body text vs big heading text).

After you've downloaded and extracted the Inter font .zip archive, open the folder and go to the "static" folder within it and grab the font variants that you actually need. For my Inter example, I will only use the following static font files:

  • Inter_18pt-Regular.ttf
  • Inter_18pt-Italic.ttf
  • Inter_18pt-ExtraBold.ttf
  • Inter_18pt-ExtraBoldItalic.ttf
  • Inter_24pt-Black.ttf (for headings)

Next, you will need to determine the Unicode ranges that you require that cover your language. I found this useful Unicode Block table to be invaluable for that. You can also just use Google or whatever search engine you prefer to look up something like "What Unicode range is Serbian Latin" and usually the search snippet will give you the correct answer (though you should always double-check!).

Tip: You can inspect any font file to see what glyphs and languages it supports with an online tool such as Font Info.

For example, if you just need everything that is present on your average US English keyboard (letters, numbers, symbols), plus some advanced punctuation and symbols, your Unicode range will be --unicodes="U+0020-007E,U+00A0-00FF,U+2000-206F". It includes:

  • U+0020-007E — Core US keyboard (includes symbols and punctuation such as !@#$%^&*()_+[]{};:'",.<>/?\|); essential
  • U+00A0-00FF — Latin 1 supplement; includes things like copyright symbol (©)
  • U+2000-206F — General punctuation; includes things like quotation marks (fancy ones), en dash (–) and em dash (—) (my beloved! ♥️). By the way, did you know that you can easily insert them in your text with a keyboard shortcut Win + - for en dash and Win + Shift + - for em dash? Nifty! ✨ 🙂

Back to my Inter example... Go to your extracted Inter .zip archive folder, and then open the Git Bash console inside the "static" folder. You will have to run the following command for every file that you intend to use, just change the file name field.

This is the full command that I use: pyftsubset Inter_18pt-Regular.ttf --output-file="inter-regular.woff2" --unicodes="U+0020-007F,U+00A0-00FF,U+0100-017F,U+0400-04FF,U+2000-206F,U+20A0-20CF,U+2100-214F" --flavor=woff2

Alternatively, if that doesn't work because of your shell configuration, try python -m fontTools.subset Inter_18pt-Regular.ttf --output-file="inter-regular.woff2" --unicodes="U+0020-007F,U+00A0-00FF,U+0100-017F,U+0400-04FF,U+2000-206F,U+20A0-20CF,U+2100-214F" --flavor=woff2

Let's break it down what each part means:

  • pyftsubset — calling the tool
  • Inter_18pt-Regular.ttf — my initial font file
  • --output-file="inter-regular.woff2" — specifying the file name and file type extension for my output file
  • --unicodes="..." — specific Unicode ranges that I need; In my case:
    • U+0020-007F — Core US keyboard
    • U+00A0-00FF — Latin 1 supplement for copyright symbol (©)
    • U+0100-017F — Latin Extended-A for Serbian Latin alphabet
    • U+0400-04FF — Standard Cyrillic alphabet for occasional text in Russian or Cyrillic Montenegrin
    • U+2000-206F — Advanced punctuation for en dash and em dash
    • U+20A0-20CF — Currency symbols; rarely used...
    • U+2100-214F — Letterlike Symbols, like trademark (™)
  • --flavor=woff2 — to specify that I want it converted to WOFF2 format

If you did everything correctly, you should see a new file appear in the same folder named inter-regular.woff2. Repeat the same process for every other file (that you need). I make one exception for the Inter_24pt-Black.ttf file. Since I only use that variant for headings, I use a much narrower --unicodes="U+0020-007E" Unicode range because I know that I only need the core US keyboard.

Adding custom-fonts to CSS stylesheet

Put your newly minted .woff2 files somewhere in your website's project folder. For example, my entire website is kept in the folder named "public", this is the one that I later upload online. Inside the "public" folder, I have a folder named "assets" and inside "assets", I have my "fonts" folder with all my .woff2 font files. You don't have to follow this specific file structure, but it's pretty standard.

Your CSS file should start with the @charset UTF-8 line and typically, all custom CSS @font-face rules come after that, at the top of the document. To import and actually use your custom fonts on your website, you will need to add the following rules in your CSS style sheet:

My own CSS code as an example (show)
/* Custom typeface — Inter (body text) */

/* Regular */
@font-face {
  font-family: "Inter";
  src: url("../fonts/inter-regular.woff2") format("woff2");
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}

/* Italic */
@font-face {
  font-family: "Inter";
  src: url("../fonts/inter-italic.woff2") format("woff2");
  font-weight: 400;
  font-style: italic;
  font-display: swap;
}

/* Extra Bold */
@font-face {
  font-family: "Inter";
  src: url("../fonts/inter-800.woff2") format("woff2");
  font-weight: 800;
  font-style: normal;
  font-display: swap;
}

/* Extra Bold Italic */
@font-face {
  font-family: "Inter";
  src: url("../fonts/inter-800-italic.woff2") format("woff2");
  font-weight: 800;
  font-style: italic;
  font-display: swap;
}

/* Black */
@font-face {
  font-family: "Inter";
  src: url("../fonts/inter-900.woff2") format("woff2");
  font-weight: 900;
  font-style: normal;
  font-display: swap;
}

Let's break each part down, line by line:

  • font-family: "Inter"; — I am giving a name to my font

  • src: url("../fonts/inter-regular.woff2") format("woff2"); — The first part is the relative path to your font file, depending on its location within your folder structure. The second one specifies what font format you are using (woff2, woff, or truetype for .ttf files).

    You can also add a fallback option, like this:

    src: ("../fonts/inter-regular.woff2") format("woff2"),
     ("../fonts/inter-regular.woff") format("woff");

    ... But it really isn't necessary unless you need to support really old browsers.

  • font-weight: 400; — specifies the weight/thickness of the font; lower number means thinner font (400 — regular, 700 — bold)
  • font-style: normal; — either normal or italic
  • font-display: swap; — usually you will use either "block", or "swap". This rule determines how your website's text will appear while your font is loading. Don't neglect that! Slower internet connections are still a thing. "Block" means that the font will be completely invisible while the custom font is loading, NOT recommended for body text, but it might work well for things like logos. "Swap" makes it so that the unstyled text will be visible during the loading time and then it will get replaced with the custom font later, after it loads. You can specify some web safe fonts that look similar to your custom font as fallback later on in the body section.

Tip: You can simulate slow network speed by pressing F12 to open developer tools and go into the "Network" tab and select one of the 2G, 3G, or 4G speed presets to throttle the connection speed. Then, perform a hard refresh by pressing Ctrl + R keys and observe.

After you added your @font-face rules, add your new font to body (or wherever you plan on using it):

body {
    font-family: "Inter", "Segoe UI", sans-serif;
}

Make sure to specify fallback fonts like I did!

Bonus typography tips

The default line-height of your website's body font is usually way too small, which leads to poor text readability, especially for some people with dyslexia. To fix this, add a line-height CSS rule to your stylesheet and start with a value that is too big, like 200% for example and then dial it down to something that looks good for your particular font. Usually, a value close to 150% looks the best.

Avoid using font weights that are too thin. They may be trendy and look sleek, but they are hard to read. Stick with font weight 400 ("regular") to stay safe. Make sure the colors have enough contrast as well.

On the other hand, it usually looks good when your headings are extra bold or even black so that they stand apart more from the main text. Make sure they are sufficiently big as well. Start with a value that is too big and then lower it down gradually until you find something that looks good. Additionally, with regard to headings, you might actually want to lower the default line-height and letter-spacing CSS values. This is not a hard rule though, so just experiment and see what looks good for your website project.

There is a nice little-known CSS rule called text-wrap. You can use text-wrap: balance; for the headings and text-wrap: pretty; for the body text. The value "balance" ensures that the text lines remain of similar length, which looks more visually pleasing. The value "pretty" prevents the formation of so-called "typographic orphans", which is a single word from a paragraph that dangles at the end of a block of text and looks unappealing. As of writing this guide, the value "pretty" doesn't work in Firefox (text just appears normal, it doesn't do anything).

Avoid using px for font sizing since it's not responsive.
